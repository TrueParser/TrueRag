# TrueRAG Ingestion Module

## Purpose
`TrueRag.Ingestion` owns canonical request normalization and ingestion execution paths.

Current scope:
- canonical contract mapping from `IngestionRequestDto`
- payload validation and fidelity detection
- async buffered ingest via WAL append + node-scoped queue publish
- sync direct ingest with bounded concurrency
- embedding-mode aware ingestion validation and execution intent capture

## Public API Surface

### DI Registration

## Queue and WAL Behavior
- Async accept path appends payload to WAL and captures coordinates.
- Async job is published to node-scoped subject: `TrueRAG.Job.Ingest.<nodeId>`.
- Queue transport uses NATS JetStream with stream provisioning warmup.
- Worker consumes queue message, rehydrates payload from WAL coordinates, persists to storage, then marks completion.
- WAL metadata and queue job carry embedding execution intent flags:
  - `RequiresInternalEmbeddingGeneration`
  - `UsesPrecomputedVectors`

## Embedding Execution Modes

- Sync path (`/api/v1/ingest/sync`):
  - strict external/precomputed mode
  - every chunk must include non-empty `Vector`
  - no internal embedding generation

- Async path (`/api/v1/ingest/async`):
  - embedding-orchestrated path
  - caller supplies chunks/text
  - vectors are generated by pipeline in worker (internal ONNX or configured external provider)
  - caller-provided vectors are rejected by contract
  - worker uses scoped descriptor (`tenant_id + app_id + collection_id`) to select provider/model
  - OpenAI is the first external provider supported in async worker orchestration
  - provider failures are mapped to deterministic errors (`auth_failed`, `rate_limited`, `timeout`, `provider_unavailable`)

Mode and descriptor rules:
- Mode resolution is scope-based: `tenant_id + app_id + collection_id`.
- Precomputed vectors are validated against active descriptor dimensions.
- Mismatch is rejected with `ingestion.embedding_space_mismatch`.

## Responsibilities
- Keep parser-facing contract mapping isolated from storage/retrieval logic.
- Keep async and sync execution on the same request contract.
- Preserve tenant/app/document scope metadata through queue and WAL boundaries.
- Preserve tenant/app/collection/document scope metadata through queue and WAL boundaries.
- Keep sync path behavior unchanged: precomputed vectors only.
- Enforce zero-trust ACL ingestion: `AllowedDocumentGroups` must contain at least one non-empty group value.
- Enforce collection scope parity: payload collection scope must match request context collection scope.
